# License System Plan

## Overview

The DENG Tool license system ties a `DENG-XXXX-XXXX-XXXX-XXXX` key to:
1. A Discord user (by Discord user ID)
2. A device (by SHA-256 of a random `install_id` — never hardware identifiers)

The system is designed to be:
- **Privacy-safe**: no IMEI, no phone number, no hardware MAC
- **Offline-first**: `LocalJsonLicenseStore` works without internet
- **Forward-compatible**: `BaseLicenseStore` interface can be swapped for Supabase

---

## Key Format

```
DENG-8F3A-B3C4-D5E6-44F0
```

- Prefix: `DENG`
- 4 groups of 4 uppercase hex digits (16 hex chars total = 64 bits of entropy)
- Total length: 19 characters

Legacy format (`DENG-<hex≥8>`) is still supported by `keystore.py` for backward compatibility. The new format is generated by `license.py`.

| Module | Purpose |
|---|---|
| `agent/license.py` | Key generation, normalization, validation, masking, hashing, install_id |

```

list_user_keys(discord_user_id)
reset_hwid(discord_user_id, key_id)
get_reset_count_24h(key_id)
get_last_seen_at(key_id)

log_license_check(**kwargs)
save_panel_config(guild_id, channel_id, message_id, updated_by)

audit_admin_action(actor_discord_user_id, action, ...)
```

## Check Result Codes

| Code | Meaning |
|---|---|
| `active` | Key is valid and device matched |
| `expired` | Key has passed its `expires_at` date |
| `revoked` | Key was revoked by an admin |
| `wrong_device` | Different `install_id_hash` is already bound |
| `not_found` | Key does not exist in the store |
| `inactive` | Key exists but status is not `active` |
| `missing_key` | No key provided at all |
| `server_unavailable` | Remote backend could not be reached |

---

## HWID Reset Rules

- **Limit**: max 5 resets per key per 24-hour rolling window
- **Active guard**: raises `ActiveKeyWarning` if `last_seen_at` was < 5 minutes ago
- Resets are logged in `reset_logs` / `hwid_reset_logs`

## Config Integration

`config.json` contains a nested `license` section:

```json
{
  "license": {
    "enabled": false,
    "mode": "local",
    "key": "",
    "server_url": "",
    "install_id": "",
    "device_label": "",
    "last_status": "not_configured",
    "last_check_at": null
  }
}
```

- `enabled=false` (default): license check is **skipped** entirely (backward compatible)
- `mode=local`: key verified via `keystore.py` / `LocalJsonLicenseStore`
- `mode=remote`: reserved for future Supabase backend (currently returns error message)

## Planned: SupabaseLicenseStore

When remote mode is implemented:
1. On device startup: call `bind_or_check_device()` → POST to Edge Function
2. Edge Function verifies key in `license_keys`, checks `device_bindings`
3. Returns result code; agent proceeds or stops
4. HWID reset via Discord panel → calls `reset_hwid()` → clears binding in Supabase

See `supabase/migrations/001_license_system.sql` for the full schema.
